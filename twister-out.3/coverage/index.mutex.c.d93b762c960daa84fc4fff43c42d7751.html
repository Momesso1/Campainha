7 /**
8 * @file
9 * @brief mutex kernel services
10 *
11 * This module contains routines for handling mutex locking and unlocking.
12 *
13 * Mutexes implement a priority inheritance algorithm that boosts the priority
14 * level of the owning thread to match the priority level of the highest
15 * priority thread waiting on the mutex.
16 *
17 * Each mutex that contributes to priority inheritance must be released in the
18 * reverse order in which it was acquired. Furthermore each subsequent mutex
19 * that contributes to raising the owning thread's priority level must be
20 * acquired at a point after the most recent "bumping" of the priority level.
21 *
22 * For example, if thread A has two mutexes contributing to the raising of its
23 * priority level, the second mutex M2 must be acquired by thread A after
24 * thread A's priority level was bumped due to owning the first mutex M1.
25 * When releasing the mutex, thread A must release M2 before it releases M1.
26 * Failure to follow this nested model may result in threads running at
27 * unexpected priority levels (too high, or too low).
28 */
